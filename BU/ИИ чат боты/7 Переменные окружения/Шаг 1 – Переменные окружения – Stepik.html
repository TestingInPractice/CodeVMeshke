2.7 Переменные окружения
1 из 1 шага пройден
Редактировать

Что такое переменные окружения?

Представь, что это «настройки» для компьютера, как параметры в игре.
Например:

  • PATH — список папок, где компьютер ищет программы (как список адресов друзей, которых ты зовёшь в гости).

  • JAVA_HOME — путь к папке с Java (чтобы программы знали, где её искать).

Зачем они нужны?

  • Чтобы программы не терялись: «Эй, компьютер, когда я пишу python, ищи его в папке C:\Python!».

  • Чтобы не повторять одно и то же: вместо длинного пути пишешь короткое имя переменной.

Пример 1: Работа с PATH (куда компьютер смотрит)

Задача: Добавить свою папку в PATH, чтобы запускать программы из неё напрямую.

Windows

  1. Создай папку C:\my_scripts и положи туда файл hello.bat с текстом:

    @echo Привет, мир!  
pause

  2. Добавь папку в PATH:

    • Через PowerShell :

      # Временное изменение (только для этой сессии)
$env:Path += ";C:\my_scripts"
# Проверь, добавилось ли:
echo $env:Path
# Запусти программу из папки:
hello.bat

    • Постоянное изменение:
      Нажми Win + S → Найди «Изменение системных переменных среды» → Добавь путь в «Переменные среды пользователя».

Linux/macOS

  1. Создай папку ~/my_scripts и файл hello.sh:

    #!/bin/bash
echo "Привет, мир!"

    Сделай файл исполняемым:

    chmod +x ~/my_scripts/hello.sh

  2. Добавь папку в PATH:

    # Временно:
export PATH=$PATH:~/my_scripts
hello.sh  # Запустится!
# Навсегда (добавь в конец файла ~/.bashrc):
echo 'export PATH=$PATH:~/my_scripts' >> ~/.bashrc
source ~/.bashrc

Пример 2: Создай свою переменную (например, GAME_DIR)

Задача: Создать переменную, которая хранит путь к папке с игрой.

Windows

# Установи переменную (временную):
$env:GAME_DIR = "C:\cool_game"
# Проверь:
echo $env:GAME_DIR
# Используй в скрипте:
cd $env:GAME_DIR

Linux/macOS

# Временная переменная:
export GAME_DIR="/home/user/cool_game"
echo $GAME_DIR
# Постоянная (добавь в ~/.bashrc):
echo 'export GAME_DIR="/home/user/cool_game"' >> ~/.bashrc

Пример 3: JAVA_HOME для работы с Java

Задача: Указать путь к Java, чтобы программы знали, где она установлена.

Windows

  1. Узнай путь к Java (обычно C:\Program Files\Java\jdk-21).

  2. Установи переменную:

    [Environment]::SetEnvironmentVariable("JAVA_HOME", "C:\Program Files\Java\jdk-21", "User")

  3. Проверь в новом терминале:

    echo $env:JAVA_HOME

Linux/macOS

export JAVA_HOME=/usr/lib/jvm/java-21-openjdk  # Путь может отличаться!
echo "export JAVA_HOME=$JAVA_HOME" >> ~/.bashrc

Системные vs Пользовательские переменные

  • Системные — как правила школы: действуют для всех.
    Пример: общий PATH для всех программ.

  • Пользовательские — как твой стиль одежды: только для тебя.
    Пример: GAME_DIR или API_KEY (ключ для доступа к чему-то личному).

Важно:

  • Если переменная не работает, перезапусти терминал!

  • Не изменяй системные переменные без надобности — можно сломать систему 😱.

Практика: Создай свой HELLO_PATH

  1. Создай папку hello_world и файл hi.txt с текстом «Я молодец!».

  2. Установи переменную HELLO_PATH, указывающую на эту папку.

  3. Напиши команду, которая выводит содержимое hi.txt через переменную:

    • Windows: Get-Content "$env:HELLO_PATH\hi.txt"

    • Linux/macOS: cat $HELLO_PATH/hi.txt

Результат: Ты только что использовал переменную окружения для доступа к файлу! 🎉

✨ Совет: Поиграй с переменными — это безопасно, если не трогать системные. Экспериментируй!
